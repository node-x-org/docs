# 积分计划（AP+RP）

#### 1. 设计初衷

在 Node-X 网络中，每一台设备、每一份算力、每一个参与者，\
都应因 **真实贡献** 而获得 **公平回报**。

因此我们建立了由 **ProofX + PoRW + AP/RP + 冷/热奖池调节** 构成的完整激励体系，\
确保“真实设备 → 真实算力 → 真实收益” 的闭环。

核心理念：\
**激励真实贡献，反对虚假繁荣。**

***

#### 2. ProofX：可信设备验证层

**ProofX** 是 Node-X 的 **可信计算验证协议**，用于确保所有节点均为真实设备，\
防止冒充、设备复用、身份伪造等 **Sybil 攻击**。

**ProofX** 验证内容包括：

| 验证维度              | 说明                      |
| ----------------- | ----------------------- |
| 设备MID（Machine ID） | 由硬件指纹、多重签名、加密唯一生成       |
| 算力真实性             | 校验CPU/GPU等性能报告与实际基准跑分一致 |
| 节点唯一性             | 防止同一设备重复注册多个身份          |
| 环境可信度             | 检测虚拟化/代理环境，验证运行隔离性      |

只有通过验证的设备，才具备参与 RP/ AP 计算与奖池分配的资格。

***

#### 3. 双积分体系：AP × RP

| 积分类型                 | 全称   | 来源                            | 核心作用        |
| -------------------- | ---- | ----------------------------- | ----------- |
| RP (Resource Points) | 资源积分 | **ProofX** + **PoRW** 验证的真实算力 | 衡量设备算力贡献    |
| AP (Action Points)   | 行为积分 | 节点任务执行、社区参与                   | 衡量参与度与生态活跃度 |

RP 代表算力贡献，AP 代表生态参与。\
两者共同构成参与者在系统中的综合贡献度，\
并最终通过智能合约映射为代币权重。

***

#### 4. 奖池与发放机制

每日系统根据网络总活跃度释放一定积分总量（初始每日 1,000,000 RP），\
并按硬件维度分配：

| 维度  | 权重  | 每日上限（RP） |
| --- | --- | -------- |
| CPU | 25% | 250,000  |
| GPU | 35% | 350,000  |
| RAM | 12% | 120,000  |
| ROM | 13% | 130,000  |
| NET | 15% | 150,000  |

奖池采用 **每半年减半机制（Halving）**，\
确保长期通胀受控、激励逐步稀缺化。

***

#### 5. ProofX 驱动的贡献评分机制

每个节点的每日有效积分计算如下：

```
有效RP = PoRW算力权重 × ProofX信誉系数 × 奖池分配比例
```

**ProofX** 的信誉系数基于设备历史验证记录、算力真实性、稳定性而动态变化。

贡献评分由以下维度组成：

| 指标                        | 说明                       |
| ------------------------- | ------------------------ |
| 性能基准 (Benchmark)          | CPU/GPU/IO/NET 实测得分      |
| 在线时长 (Uptime)             | 连续在线率                    |
| 稳定性 (Stability)           | 系统运行波动与故障率               |
| 任务完成度 (Completion Rate)   | 指令执行/任务成功比例              |
| 验真比率 (R, from **ProofX**) | **ProofX** 验真通过率，用于判断真实性 |

所有指标通过 EMA 平滑处理，长期稳定节点将获得更高信誉权重。

***

#### 6. 冷池与热池动态调节机制

为维持生态平衡并防止算力集中，系统引入 **冷/热奖池调节机制**。

**6.1 分类逻辑**

* **热池 (Hot Pool)**：\
  实际算力贡献率 ≥ 预期权重（竞争活跃），可获得额外奖励。\
  来自冷池的部分未发放奖励可转入热池，最高溢出上限为原奖池的 50%。
* **冷池 (Cold Pool)**：\
  实际算力贡献率 < 预期权重（参与不足），\
  奖励按比例削减，差额转移至热池。

**6.2 ProofX参与逻辑**

**ProofX**会实时评估各节点的真实性得分（R），\
低R节点（疑似虚假贡献）将自动归入冷池，\
高R节点将获得热池优先奖励。

因此冷/热池不仅基于算力贡献，还结合设备可信度形成 **双维动态平衡**。

**6.3 结果**

* 奖励随真实活跃度自动调节，控制通胀；
* 稀缺且高信誉的算力获得额外激励；
* 网络整体保持公平、稳健的算力分布。

***

#### 7. 社区参与与 AP 激励

AP（行为积分）主要奖励以下生态行为：

* 节点首次接入并通过 **ProofX** 验证；
* 完成任务、提交指令；
* 邀请新节点接入；
* 治理投票、质押、反馈等行为。

系统会定期根据 **AP × RP × ProofX信誉系数**，\
计算最终代币分配权重，确保既奖励算力，也奖励活跃贡献。

***

#### 8. 积分 → 代币映射机制

积分本身不具备货币属性，\
但通过链上智能合约可按周期映射为代币权重。

映射原则：

* **公平性**：映射比例与综合积分成正比；
* **验证优先**：未通过 **ProofX / PoRW** 的节点无资格映射；
* **持续性**：长期活跃节点获得衰减保护；
* **反作弊**：风控标记节点积分将被冻结。

***

#### 9. 总结

| 模块         | 功能         | 激励来源          |
| ---------- | ---------- | ------------- |
| **ProofX** | 验证设备真实性    | 防止女巫攻击、建立可信身份 |
| **PoRW**   | 验证任务真实性与质量 | 动态积分分配        |
| AP 行为模型    | 激励真实生态参与   | 节点任务、治理、互动    |
| 冷/热池调节     | 动态平衡资源激励   | 稀缺算力激励加成      |
| 积分→代币映射    | 构建经济闭环     | 公平分配长期贡献      |

通过 **ProofX+PoRW+ AP + RP + 冷/热奖池调节机制**，\
Node-X 实现了从设备验证到经济激励的全流程信任闭环：

> “Make Any Device On-Chain, Make Compute Social.”
