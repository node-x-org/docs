# 積分計劃(AP+RP)

<p align="right"><a href="https://docs.node-x.xyz/en/product-manual/nodehub/points-program-ap+rp">English</a></p>

#### 1. 設計初衷

在 NodeX 網絡中,每一台設備、每一份算力、每一個參與者,都應因**真實貢獻**而獲得**公平回報**。

因此我們建立了由 **ProofX + PoRW + AP/RP + 冷/熱獎池**調節 構成的完整激勵體系,\
確保「真實設備 → 真實算力 → 真實收益」的閉環。

核心理念:\
**激勵真實貢獻,反對虛假繁榮。**

***

#### 2. ProofX：可信設備驗證層

ProofX 是 NodeX 的**可信計算驗證協議**,用於確保所有節點均為真實設備,防止冒充、設備複用、身份偽造等 **Sybil 攻擊**。

**ProofX** 驗證內容包括:

| 驗證維度              | 說明                      |
| ----------------- | ----------------------- |
| 設備MID(Machine ID) | 由硬體指紋、多重簽名、加密唯一生成       |
| 算力真實性             | 校驗CPU/GPU等性能報告與實際基準跑分一致 |
| 節點唯一性             | 防止同一設備重複註冊多個身份          |
| 環境可信度             | 檢測虛擬化/代理環境,驗證運行隔離性      |

只有通過驗證的設備,才具備參與 RP 計算與獎池分配的資格。

***

#### 3. 雙積分體系：AP × RP

<table><thead><tr><th width="162">積分類型</th><th>全稱</th><th>來源</th><th>核心作用</th></tr></thead><tbody><tr><td>RP (Resource Points)</td><td>資源積分</td><td><strong>ProofX</strong> + <strong>PoRW</strong> 驗證的真實算力</td><td>衡量設備算力貢獻</td></tr><tr><td>AP (Action Points)</td><td>行為積分</td><td>節點任務執行、社區參與</td><td>衡量參與度與生態活躍度</td></tr></tbody></table>

RP 代表算力貢獻,AP 代表生態參與。\
兩者共同構成參與者在系統中的綜合貢獻度,並最終通過智能合約映射為代幣權重。

***

#### 4. 獎池與發放機制

每日系統根據網絡總活躍度釋放一定積分總量(初始每日 1,000,000 RP),並按硬體維度分配:

| 維度  | 權重  | 每日上限(RP) |
| --- | --- | -------- |
| CPU | 25% | 250,000  |
| GPU | 35% | 350,000  |
| RAM | 12% | 120,000  |
| ROM | 13% | 130,000  |
| NET | 15% | 150,000  |

獎池採用**每半年減半機制(Halving),**&#x78BA;保長期通脹受控、激勵逐步稀缺化。

***

#### 5. ProofX 驅動的貢獻評分機制

每個節點的每日有效積分計算如下:

```
有效RP = PoRW算力權重 × ProofX信譽係數 × 獎池分配比例
```

**ProofX** 的信譽係數基於設備歷史驗證記錄、算力真實性、穩定性而動態變化。

貢獻評分由以下維度組成:

| 指標                        | 說明                       |
| ------------------------- | ------------------------ |
| 性能基準 (Benchmark)          | CPU/GPU/IO/NET 實測得分      |
| 在線時長 (Uptime)             | 連續在線率                    |
| 穩定性 (Stability)           | 系統運行波動與故障率               |
| 任務完成度 (Completion Rate)   | 指令執行/任務成功比例              |
| 驗真比率 (R, from **ProofX**) | **ProofX** 驗真通過率,用於判斷真實性 |

所有指標通過 EMA 平滑處理,長期穩定節點將獲得更高信譽權重。

***

#### 6. 社區參與和 AP 激勵

AP(行為積分)主要獎勵以下生態行為:

* 完成任務、提交指令;
* 邀請新節點接入;
* 治理投票、質押、反饋等行為。

系統會定期根據 **AP × RP × ProofX信譽係數**,計算最終代幣分配權重,確保既獎勵算力,也獎勵活躍貢獻。

***

#### 7. 積分 → 代幣映射機制

積分本身不具備貨幣屬性,\
但通過鏈上智能合約可按週期映射為代幣權重。

映射原則:

* **公平性**：映射比例與綜合積分成正比;
* **驗證優先**:未通過 ProofX / PoRW 的節點無資格映射;
* **持續性:**&#x9577;期活躍節點獲得衰減保護;
* **反作弊:**&#x98A8;控標記節點積分將被凍結。

***
